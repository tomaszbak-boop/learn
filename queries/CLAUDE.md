# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is an e-commerce data utilities project that provides query functions for a SQLite database (`ecommerce.db`). The project uses TypeScript with the `sqlite` promise-based wrapper over `sqlite3`.

## Database Schema

The SQLite database contains tables for a complete e-commerce system including:

- customers, addresses, customer_segments, customer_activity_log
- products, categories, inventory, warehouses
- orders, order_items
- reviews
- promotions

See `src/schema.ts` for the complete schema. Run `src/main.ts` to initialize the database.

## Development Commands

```bash
npm run setup       # Install dependencies and generate .claude/settings.local.json from template
npm run sdk         # Run sdk.ts (Claude Agent SDK example)
```

## Working with Queries

All query functions are `async` and use the `sqlite` promise-based wrapper — `db.get()` and `db.all()` return Promises directly. No callbacks needed.

```typescript
import { Database } from "sqlite";

export async function getCustomerByEmail(
  db: Database,
  email: string,
): Promise<any> {
  return await db.get(`SELECT * FROM customers WHERE email = ?`, [email]);
}
```

- Single record: `db.get()` — returns one row or `undefined`
- Multiple records: `db.all()` — returns an array
- Always use parameterized queries (`?` placeholders)

## Hooks

`.claude/settings.local.json` (generated by `npm run setup` from `.claude/settings.example.json`) configures two hooks:

- **PreToolUse on Write/Edit**: `hooks/query_hook.js` — uses the Claude Agent SDK to check if new query functions in `src/queries/` duplicate existing ones; blocks with exit code 2 if duplication is detected
- **PostToolUse on Write/Edit**: runs `prettier` on the modified file, then `hooks/tsc.js` for type checking

## Critical Guidance

- All database queries must be written in `./src/queries/`
- Check existing query functions before adding new ones — the pre-tool hook enforces this automatically
